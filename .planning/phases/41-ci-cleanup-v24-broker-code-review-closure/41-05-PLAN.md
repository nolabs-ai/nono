pub fn set_all(vars: &[(&'static str, &str)]) -> Self {
    let original = vars
        .iter()
        .map(|(key, _)| (*key, std::env::var(key).ok()))
        .collect::<Vec<_>>();

    for (key, value) in vars {
        std::env::set_var(key, value);
    }

    Self { original }
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test process global env vars | Parallel cargo tests share process-global env state. Mutating shared keys (PATH, PATHEXT, etc.) without a Drop-restore guard creates a race where the test under fix sees a different baseline than the one it asserts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-05-01 | T (Tampering) | If `EnvVarGuard::set_all` is invoked without a corresponding `let _env = ...` binding to keep the guard alive for the test duration, the guard's Drop fires immediately and the restore-then-mutate race appears. | mitigate | The Pattern in this plan binds `let _env = ...` explicitly; the `#[must_use]` attribute on `set_all` (verified at test_env.rs:27) makes the compiler warn if the result is discarded. |
| T-41-05-02 | I (Information Disclosure) | LOW — the canonical PATH/PATHEXT values pinned here are public Windows defaults; no secret data crosses the boundary. | accept | LOW severity. |
</threat_model>
