## Unregistered Flags

None. Plan 01 § "Threat Surface Scan", Plan 02, and Plan 03 SUMMARYs explicitly state "No threat flags raised". The implementation introduces exactly one new env var (`NONO_TEST_HOME`) and zero new file I/O, schema, or endpoint surface. All exposed surface maps to threats T-27.1-01..14.

## Accepted Risks Log

The following dispositions are explicitly accepted in the threat model and documented here for forensic record:

1. **T-27.1-02** — `NONO_TEST_HOME` directs file I/O to attacker-controlled path. Same risk profile as existing `HOME`/`USERPROFILE`. Forensic warn-once provides audit trail.
2. **T-27.1-03** — `tracing::warn!` of resolved override path. Log destination is user-controlled; no PII or secret material; path was already known to whoever set the env var.
3. **T-27.1-07** — `session.rs:410` discards `EnvVarValidation` discriminant in favor of `ConfigParse` (legacy contract). Validation error still fires at 14 other callsites.
4. **T-27.1-08** — Display-only callsites silently swallow `EnvVarValidation` via `if let Ok(home) = ...`. UX requirement; non-display callsites still surface the error in the same process.
5. **T-27.1-10** — Internal error-mapping logic visible in `session.rs:410` `.map_err` expression. Standard Rust practice; non-executable from a security perspective.
6. **T-27.1-12** — `tempfile::TempDir` cleanup is best-effort; test artifacts may persist on disk on cleanup failure. Test-only data, no real secrets.
7. **T-27.1-13** — Test panic could leak env-var state. Per-invocation suffix prevents collisions; `cargo test` isolates per-thread.

## Deviations & Status

- **Phase 27.1 Plan 03** triggered the D-27.1-14 large-fix contingency: re-#[ignore]'d both audit-attestation tests (`febdc73d`); the seam itself was the deliverable, verified end-to-end on Windows host. v2.4-FU-1 (audit-loader swap) + v2.4-FU-2 (bundle-target architecture) landed in Phase 27.2 plans 27.2-01 / 27.2-02 / 27.2-04 (`0a4aa279`), removing the `#[ignore]` attributes. Phase 27.1 acceptance is independent of this follow-up.
- **Plan 02 clippy auto-fix:** Display callsites use `if let Ok(home) = ...` instead of plan-stated `if let Some(home) = ....ok()`. Newer rust-1.95 clippy flags `.ok()` as `match_result_ok` redundant; the substitution is semantically identical and preserves the silent-swallow display-path intent. Documented in 27.1-02-SUMMARY.md "Auto-fixed Issues".
