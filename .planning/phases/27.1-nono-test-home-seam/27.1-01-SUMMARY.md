# Phase 27.1 Plan 01: NONO_TEST_HOME Seam Foundation Summary

Add nono_home_dir() helper plus extend user_state_dir() in crates/nono-cli/src/config/mod.rs to honor NONO_TEST_HOME env-var override (absolute-path-validated, warn-once on first use, fail-closed on relative); locks 4 unit tests for the seam contract independent of Phase 27 integration tests.

## What Shipped

### Helper: nono_home_dir() -> Result<PathBuf>

Module location: crates/nono-cli/src/config/mod.rs (extended; immediately after validated_tmpdir()).

- When NONO_TEST_HOME is set to an absolute path: returns that path verbatim.
- When NONO_TEST_HOME is set to a non-absolute path: returns NonoError::EnvVarValidation with var "NONO_TEST_HOME" and reason "must be an absolute path, got: ..." (fail-closed, no silent fallthrough).
- When NONO_TEST_HOME is unset: falls through to dirs::home_dir().ok_or(NonoError::HomeNotFound) - byte-identical to status quo for the 10 standard ?-shape callsites that Plan 02 will migrate.
- On first successful override resolution: emits exactly one tracing::warn! line "NONO_TEST_HOME override active: {path}" per process (guarded by static WARNED: std::sync::OnceLock<()> per policy.rs:1235 analog).

Helper signature pre-bakes the ? for the 10 standard dirs::home_dir().ok_or(NonoError::HomeNotFound)? callsites; migration in Plan 02 is a mechanical replacement.

### Extension: user_state_dir() -> Option<PathBuf> shape (D-27.1-05/06)

Same module. Function signature unchanged (Option<PathBuf> preserved per D-27.1-05 invariant b).

- When NONO_TEST_HOME is set to an absolute path: returns Some(<NONO_TEST_HOME>/.nono). The .nono segment mirrors production layout, so audit_root() (.../.nono/audit) and rollback_root() (.../.nono/rollbacks) co-locate under one parent - closes Phase 27 Blocker 2 path-mismatch class on Windows (D-27.1-06).
- When NONO_TEST_HOME is unset: byte-identical to status quo - dirs::state_dir().or_else(dirs::data_local_dir).map(|p| p.join("nono")).
- When NONO_TEST_HOME is set but non-absolute: falls through to platform default (intentional; the actual fail-closed surface is nono_home_dir() - see Decisions).
