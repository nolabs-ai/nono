## Decisions Made

- **Picked Option 1 (minimal-diff) per CONTEXT § Claude's Discretion**, as the plan instructed. The plan's interfaces block stated snapshot creation uses a separate variable (not `AuditState.session_dir`); execution-time grep showed two `SnapshotManager::new` call sites consuming `audit_state.session_dir`. Resolved by adding the sibling `rollback_dir` field rather than promoting `audit_session::ensure_rollback_session_dir` or doing a wider refactor — keeps the change scoped to one file and preserves the planning artifact's `files_modified` list.
- **Audit-only `initialize_audit_snapshots` continues to use `session_dir` (== audit dir).** In audit-only mode `rollback_dir` is `None` and the snapshot manager only computes a merkle root (no persisted `changes/` directory), so the path it sees is irrelevant to the must_haves.
- **Audit-events ledger co-locates with bundle.** `supervised_runtime.rs:244`'s `AuditRecorder::new(state.session_dir.clone())` was a passive consumer of `session_dir`; with `session_dir` repointed, the ledger now lands at `<audit_root>/<id>/audit-events.ndjson` instead of `<rollback>/<id>/audit-events.ndjson`. This is consistent with the bundle-ledger trust pairing and was implicit in must_have #4 (`AuditState.session_dir` always points at audit-root-rooted path when `audit_integrity_requested`).
