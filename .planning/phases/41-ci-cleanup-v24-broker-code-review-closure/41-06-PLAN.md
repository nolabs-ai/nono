<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Broker argv ↔ broker process | The broker process receives argv from a parent (`nono-cli`'s supervisor or, if invoked directly, the OS). Malformed or hostile argv crosses this trust boundary. The broker is Medium-IL; its argv parser is the only sanitization gate before raw HANDLE values flow into `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` setup and `UpdateProcThreadAttribute` system calls. |
| Rust library ↔ C FFI consumers | nono-py + nono-ts + any other C-API consumer reads `NonoErrorCode` integer values returned via `map_error`. Wrong semantic mapping leaks "this is a user path issue" semantics to consumers who interpret `-1 (ErrPathNotFound)` as "user-typo their file path" when the actual cause is "broker installation defect (sandbox cannot init)". |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-41-06-01 (HIGH severity) | T (Tampering) | Broker accepts `--inherit-handle 0x0` (null HANDLE) without validation. Passing null HANDLE to `PROC_THREAD_ATTRIBUTE_HANDLE_LIST` is undefined Win32 behavior; pseudo-handle confusion at `(HANDLE)0` could resolve to the calling process's pseudo-handle in some Win32 paths, exposing parent process handles to the spawned child. Per CR-02 todo: "Defense-in-depth against malformed broker invocations." | mitigate | CR-02 reject in argv parser at `crates/nono-shell-broker/src/main.rs:87-99`. Reject `raw_value == 0` AND `raw_value == usize::MAX` (== `INVALID_HANDLE_VALUE` on the pointer width). Tested by new `parse_args_null_inherit_handle_returns_error` test (Wave 0). |

Per CR-03 todo + Plan 31 verifier note: `UpdateProcThreadAttribute(PROC_THREAD_ATTRIBUTE_HANDLE_LIST, lpValue=ptr, cbSize=0)` returns `ERROR_BAD_LENGTH` at runtime — but per CR-03 todo option (b), an alternative implementation could use plain `STARTUPINFOW` (default-inherit), which would inherit ALL inheritable handles in the parent process.
