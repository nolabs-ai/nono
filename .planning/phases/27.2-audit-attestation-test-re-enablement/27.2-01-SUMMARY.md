## Decisions Made

- **Migrate cmd_show alongside cmd_verify (D-27.2-04 cascade):** Planner's interfaces block stated "cmd_verify (line 608) only uses load_session" — this turned out to be incorrect. `cmd_show` (line 351) also called `load_session`. Per CONTEXT D-27.2-04 ("always-audit-first; no conditional-on-flags loader picking"), the right fix is for `cmd_show` to ALSO use the audit-aware loader so `nono audit show <id>` works on audit-only sessions. Logged as Rule 3 deviation (blocking issue: leaving cmd_show on rollback_session::load_session would still compile but flip its semantics in the wrong direction; switching cmd_show too is the correct fix per phase intent).
- **One-site SessionInfo conversion over helper-signature change:** `print_show_json`, `session_status_label`, `group_by_project` are called from BOTH `cmd_show` (now audit_session::SessionInfo) AND `cmd_list`/`cmd_cleanup` (still rollback_session::SessionInfo via discover_sessions). Changing helper signatures would cascade into `cmd_list`'s discover_sessions return type → out-of-plan-scope per CONTEXT D-27.2-04 + Boundary "no regressions on rollback-only and dual-target sessions". The conversion shim is the minimal-diff bridge.
- **Warning text wording verbatim from CONTEXT § Specifics:** The format string is locked in CONTEXT.md ("audit-attestation bundle found at legacy {} path; canonical location is {}. This compatibility shim is removed in v2.5 -- re-sign affected sessions if needed."). No re-wording.
